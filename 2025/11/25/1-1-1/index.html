十进制0.1的二进制无限循环原理

[双子星的讯息]https://www.lanqiao.cn/problems/20556/learning/?page=1&first_category_id=1
AC代码
#include<bits/stdc++.h>
using namespace std;
int main()
{
int res=0;
for(long long i=sqrt(10244201);i<=10000000;i++)
{
long long t=i*i+10011001;
if(abs(sqrt(t)-round(sqrt(t)))<1e-12) res++;
}
cout<<res;
return 0;
}
十进制整数转二进制用“除2取余”，而小数转二进制用 “乘2取整，顺序排列”，具体步骤：

1. 用小数部分 × 2，取出整数部分（0或1），作为二进制小数的下一位；
2. 用剩余的小数部分（乘2后去掉整数部分的结果）重复步骤1；
3. 直到小数部分为0（有限小数），或进入无限循环（无限循环小数）；
4. 把取出的整数部分按顺序排列，就是二进制小数。

本质是 进制的“基数互质”问题：

  • 十进制的基数是10（质因数：2×5），二进制的基数是2（质因数：2）；
  • 一个十进制有限小数能转换成二进制有限小数的条件是：该小数的分母（最简分数形式）只含2的质因数（因为二进制只有质因数2）；
  • 而0.1的最简分数是1/10，分母10=2×5，含有二进制没有的质因数5 → 无法用二进制有限小数表示，只能变成无限循环小数。

就像十进制中1/3=0.333…无限循环（因为10不含质因数3），二进制中1/10（即0.1）也会因分母含5而无限循环。

计算机存储浮点数（如C语言中的 double ）时，只能用有限位数的二进制表示（ double 约占8字节，64位），无法存储无限循环的二进制小数。因此：

  • 计算机只能近似存储0.1的二进制（截断无限循环部分）；
  • 这种近似会导致微小误差（比如0.1在计算机中实际存储为≈0.10000000000000001）；
  • 这也是之前判断完全平方数时，不能直接用 == 比较浮点数，而要用 1e-12 作为阈值的核心原因——避免被这种微小误差误导。

对于水瓶赊账问题问题

每  m  个空瓶能换 1 瓶水，相当于每消耗  m-1  个空瓶，就能额外喝到 1 瓶水（因为换的那瓶水喝完后，会再产生 1 个空瓶，抵消了“换瓶”消耗的 1 个空瓶）。

因此，最多能喝的瓶数 = 初始瓶数  n  + 用空瓶额外换的瓶数（即  n // (m-1)）
